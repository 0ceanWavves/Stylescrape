Button Components

Prism Financial's button system provides consistent, reusable button patterns that maintain visual coherence throughout the interface.

Primary Button
<button class="btn btn-primary">Primary Button</button>

Used for primary actions that we want users to focus on. Limited to one primary button per screen.

Secondary Button
<button class="btn btn-secondary">Secondary Button</button>

Used for secondary actions that are still important but shouldn't compete with the primary action.

Text Link
Text Link
<a href="#" class="link">Text Link</a>

Used for navigational actions and less important actions that don't warrant a full button.

Button Variations

Button with Icon
<button class="btn btn-primary"> <svg>...</svg> Button with Icon </button>
Small Button
<button class="btn btn-primary btn-sm">Small Button</button>
Disabled Button
<button class="btn btn-primary" disabled>Disabled Button</button>

Implementation Guidelines

  • Use primary buttons sparingly - only one per major section or form
  • Maintain consistent padding and sizing across button types
  • Button text should be clear and action-oriented
  • Icons should enhance meaning, not replace text
  • Use disabled state only when the action is temporarily unavailable

Buttons should always provide clear visual feedback on hover and click states to indicate interactivity.
